PROJETO: PADRONIZACAO E CLASSIFICACAO DE CSV COM PANDAS

OBJETIVO

Criar um script simples em Python (app.py) para ler um arquivo CSV, criar/garantir colunas, aplicar regras de classificacao e gerar um novo arquivo tratado.

O foco e simplicidade, clareza e facilidade de inspecao.

NAO usar codigo avancado. NAO usar funcoes complexas desnecessarias.

----------------------------------------

ESTILO DE CODIGO

- Codigo deve ser simples e didatico
- Usar prints explicativos em cada etapa
- Evitar funcoes complexas
- Evitar encadeamento de metodos
- Fazer passo a passo (uma acao por vez)
- Variaveis com nomes claros
- Codigo deve parecer "falando com o usuario"

----------------------------------------

FLUXO DO PROJETO

1. Ler arquivo CSV
2. Mostrar quantidade de linhas
3. Garantir colunas obrigatorias
4. Mostrar quais colunas foram criadas
5. Aplicar regras (condicoes simples usando df.loc)
6. Medir alteracoes feitas
7. Gerar arquivo novo (NUNCA sobrescrever o original)

----------------------------------------

ENTRADA E SAIDA

Entrada:
- data/5_estrelas_fevereiro.csv

Saida:
- data/5_estrelas_fevereiro_tratado.csv

NUNCA sobrescrever o arquivo original.

----------------------------------------

INSPECAO (MUITO IMPORTANTE)

Para cada coluna criada ou alterada, o sistema deve mostrar:

- Quantidade de linhas total
- Quantidade de valores preenchidos ANTES
- Quantidade de valores preenchidos DEPOIS
- Quantidade de valores alterados
- Quantidade de valores nao alterados

Exibir tudo com prints claros.

Exemplo:

[CLASSIFICACAO]
Total de linhas: 1000
Antes preenchido: 0
Depois preenchido: 850
Alterados: 850
Nao alterados: 150
Total de combinacoes:

----------------------------------------

REGRAS

Todas as regras devem ser feitas no app.py usando:

df.loc[condicao, 'coluna'] = valor

NAO usar funcoes externas complexas.
NAO usar apply no inicio (somente se extremamente necessario).

----------------------------------------

ESTRATEGIA

O projeto sera feito por partes:

1. Primeiro: garantir colunas
2. Depois: preencher uma coluna por vez
3. Depois: adicionar regras mais complexas

Sempre testando e validando antes de avancar.

----------------------------------------

PRINCIPIOS IMPORTANTES

- Clareza > performance
- Simples > sofisticado
- Inspecionavel > automatico
- Seguro (nao perder dados)

----------------------------------------

OBJETIVO FINAL

Criar um pipeline simples, confiavel e facil de entender, que possa ser evoluido aos poucos sem perder controle.
